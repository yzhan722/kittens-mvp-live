# 前期留存数值 · v0.40.0

## 诊断

- 树果田有效产量过低（×0.5 终算后约 16s/个）
- 无 `timeSec` 的研究公式常算出 ≥90s，再被保底 60s 卡住
- 新档从 0 田、0 树果开始，第一口粮全靠点采集

## 改动

| 项 | 前 | 后 |
|----|----|----|
| 研究时间 | 公式易≥90s，保底60 | 小成本 soft-cap≤25s；核心科技短 timeSec |
| 精灵球基础 | 20×2≈40 | 15×2≈30，4s |
| 树果田 | 0.0625/n | 0.15/n |
| 新档 | 0田0果 | 1田+12树果；未抓存档一次性补齐 |
| 采集 | 起步+1 | 起步+3，档位更肥 |
| 灌溉门槛 | 田≥3 | 田≥2 |

后期高 `eq` / `timeMul` 链基本不动。
